矢量处理
会话流程

U
输出叫 roads_out.json
A
脚本预览：
@echo off
ogr2ogr -f "GeoJSON" 'roads_out.json' 'roads.shp'
REM Done
确认执行？(Y/N)
U
Y
A
执行完成！输出: roads_out_20260526_143052.json
IDLE → PARAM_COLLECT → SCRIPT_PREVIEW → EXECUTING → IDLE
需求映射
F2 意图分类：自然语言 → 模板 ID（shp2geojson）
F3 参数抽取：提取 input、output 路径，追问缺失字段
F4 Jinja2 模板渲染生成 .bat 脚本
F5 先展示脚本内容，严格 Y/N 确认后执行
F8 会话内记忆：多轮对话保留上下文（已知任务不重复询问）
P1 模板化命令：ogr2ogr 命令通过 Jinja2 模板渲染，禁止字符串拼接
P2 先展后行：必须完整展示脚本并获得确认
P3 沙箱：cwd 限定工作空间，输出文件加时间戳防覆盖

场景四：GDAL 工具文档问答
文档检索
会话流程
U
ogr2ogr 能输出哪些格式？
A
基于 GDAL 官方文档，ogr2ogr 支持以下输出格式：
• GeoJSON / GeoJSONSeq
• ESRI Shapefile
• GeoPackage
• KML / CSV / SQLite...
完整列表请参见文档。
IDLE → RAG 检索 → LLM 生成回答 → 保持 IDLE
需求映射
F1 文档问答：基于本地 GDAL HTML 文档，通过 ChromaDB 向量检索
P4 仅本地文档：RAG 只检索本地官方文档，不混合网络内容
AC-1 中英跨语言检索：中文问题匹配英文文档片段（多语言 Embedding）
F11 Agents.md 内容注入系统提示词，提供项目级记忆
