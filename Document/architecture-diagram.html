🔄 典型数据流（成功路径）

1
用户输入自然语言
CLI 层 REPL 读取 "把 roads.shp 转成 GeoJSON" → 传递给 SessionProcessor.process()
2
意图分类
LLM 层 classify_intent() → 高置信度(0.95)匹配 shp2geojson 模板 → 状态: IDLE → PARAM_COLLECT
3
参数收集与校验
多轮对话中 extract_params() 提取参数 → ParamValidator 链校验 (路径安全/格式/EPSG) → 追问缺失字段
4
脚本渲染与安全校验
TemplateEngine.render() → Jinja2 + quote/safe_path 过滤器 → ScriptSecurityChecker 二次正则检查
5
脚本展示与用户确认
状态: SCRIPT_PREVIEW → 展示完整脚本 → 严格 Y/N 确认 (dry-run 模式跳过执行)
6
沙箱执行
ScriptExecutor.execute() → subprocess.run(cwd=workspace.root, timeout=300) → 返回 ExecutionResult

🚫 安全红线 (Security Principles)

P1 — 模板化命令，杜绝幻觉
所有 GDAL 命令必须通过 Jinja2 模板渲染生成，严禁动态字符串拼接。LLM 只负责选择模板 ID 和填充参数。
P2 — 先展后行
任何脚本必须向用户完整展示内容，获得明确的 Y/N 确认后才可执行。不可跳过确认环节。
P3 — 最小权限
默认输出到工作空间，输出文件自动加时间戳防覆盖。路径经规范化后使用。workspace v2.0 是记忆锚点，不是安全边界。
P4 — 模板知识唯一
用法指导类知识仅来源于 J2 模板元数据，基础概念类回答由 LLM 参数知识提供，禁止调用外部 API。
P5 — 最小依赖
生产依赖仅锁定 anthropic、jinja2 两个库。零额外运行时依赖，降低供应链攻击面。
