矢量 GDAL / ogr2ogr

Shapefile 转 GeoJSON

将 Shapefile 格式矢量数据转换为 GeoJSON，支持指定坐标系。使用 ogr2ogr 工具执行。

参数

2/3 已填写
input t_srs output
input * 必填 file_path
输入 Shapefile 路径（.shp 及其 .shx, .dbf 需同目录）
output * 必填 file_path
输出 GeoJSON 文件路径
默认: 工作区/output/<name>_<timestamp>.geojson
t_srs crs
目标坐标参考系统
默认: EPSG:4326 (WGS84)

概念与注意事项

💡
GeoJSON 默认使用 EPSG:4326 (WGS84)，这是 Web 地图标准坐标系
⚠️
输入文件必须包含 .shp 主文件及其关联的 .shx, .dbf 文件
📌
中文属性名可能在某些软件中出现乱码，建议提前处理编码

常见错误

✕
Unable to open datasource — 检查 .shp 关联文件是否齐全
✕
Failed to create file — 输出目录没有写入权限

#!/bin/bash
# Pipeline: shp2geojson → clip_raster → gdal2tiles
# Step 1: Shapefile 转 GeoJSON
ogr2ogr -f "GeoJSON"
  ${STEP1_OUT}
  /data/beijing.shp
  -t_srs EPSG:4326

# Step 2: 栅格裁剪
gdalwarp -cutline ${STEP1_OUT}
  -crop_to_cutline
  /data/dem.tif
  ${STEP2_OUT}

# Step 3: 生成瓦片
gdal2tiles.py -z 5-15
  ${STEP2_OUT}
  ${OUTPUT_DIR}
